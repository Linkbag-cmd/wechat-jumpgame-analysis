3).bottle report 
该数据包的作用是把游戏过程中的关键事件+最终统计数据上报给服务器，相当于总结性报告：
<img width="996" height="936" alt="屏幕截图 2026-03-04 135202" src="https://github.com/user-attachments/assets/25d75e97-539a-4506-947b-0305f09359a1" />

这里的report_list 记录的是游戏启动时间，初始化准备以及最终游戏结束时的上报。可以看到上报时有分数，历史最佳，游戏时间，尝试次数和破纪录时的记录。即在跳一跳的完整对局流程中，wxagame_bottlereport 接口承担了“行为统计上报”职责。其 report_list 数组会分阶段上报游戏事件（type 35/10 为启动标记，type 2 为终局总结）。终局条目中包含 score、best_score、break_record、duration、using_prop 等核心统计，与 settlement 接口的 action-data 形成双重校验。这是腾讯防止“只改分数不改行为”的重要机制之一。
